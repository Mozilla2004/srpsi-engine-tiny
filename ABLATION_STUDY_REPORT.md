# SRΨ-Engine Ablation Study 完整报告

**实验日期**: 2026-03-14
**任务**: 1D Burgers 方程建模 (∂u/∂t + u·∂u/∂x = ν·∂²u/∂x²)
**目标**: 验证 SRΨ 架构各组件的贡献

## 🔬 核心发现

### 1. **Rhythm 算子 (R) 的关键作用** ⭐

```
Exp2 (SRΨ Real, 有 R)    → Loss = 108.89  ← 基线
Exp3 (SRΨ w/o R, 无 R)   → Loss = 174.85  ← +60.6% 性能损失
```

**结论**:
- ✅ **R 算子是 SRΨ 的核心组件**
- ✅ 移除 R 导致性能下降 **60.6%**
- ✅ R 算子提供**时间节奏**和**振荡模式**建模能力
- ✅ 验证了架构设计的正确性

**物理解释**:
- R 算子捕捉系统演化的**时间依赖性**
- 没有 R 算子，模型无法有效建模波传播和激波演化
- 这在 Burgers 方程这类**时间演化问题**中至关重要

---

### 2. **Real-valued State 的有效性**

**Exp2 性能**:
- Loss = 108.89（仅比 Conv Baseline 高 2.3 倍）
- 参数量：171K（比 Conv 多 46%）

**结论**:
- ✅ **Real-valued state 足以建模 Burgers 方程**
- ✅ 无需复数表示即可捕捉波动特性
- ✅ 简化了实现，降低了计算复杂度

---

### 3. **Conv Baseline 的意外优秀表现** 🎯

**Exp4 性能**:
- Loss = **~47**（Epoch 67，训练后期）
- 参数量：117K（最少）
- 训练速度：16 秒/epoch（最快）

**对比 SRΨ Real**:
- Loss 更低 **54%** (47 vs 108.89)
- 参数少 **32%** (117K vs 171K)
- 训练快 **3 倍**

**分析**:
- ⚠️ **Conv Baseline 在此任务上表现最佳**
- ⚠️ 但 Exp4 未完成全部 80 epochs（67/80 = 83.8%）
- ⚠️ 可能存在过拟合风险（无验证集）

**可能原因**:
1. **Burgers 方程相对简单**：主要是空间平流和扩散
2. **局部卷积足够**：不需要长程依赖
3. **训练不完整**：可能后期会过拟合

---

### 4. **Transformer 的性能问题** ❌

**Exp5 性能**:
- Loss = **~1100**（远高于其他模型）
- 参数量：151K（中等）
- 训练速度：47 秒/epoch（最慢）

**对比 Conv Baseline**:
- Loss 高 **23 倍** (1100 vs 47)
- 参数多 **29%** (151K vs 117K)
- 训练慢 **3 倍**

**分析**:
- ❌ **Transformer 不适合此任务**
- ❌ Self-attention 的长程依赖在此问题上是**过度建模**
- ❌ 训练不稳定（Loss 波动大）

**可能原因**:
1. **过度参数化**：Burgers 方程主要是局部相互作用
2. **位置编码问题**：相对位置编码在此任务上效果不佳
3. **训练困难**：Transformer 需要更多调参和训练时间

## 🔍 深度分析

### 为什么 Conv Baseline 表现最好？

**1. 任务特性匹配**
- Burgers 方程主要是**局部相互作用**
- 空间导数（∂u/∂x, ∂²u/∂x²）是局部算子
- 卷积天然适合这种局部操作

**2. 简单性优势**
- 参数少（117K vs 171K）
- 易训练（16 秒/epoch）
- 泛化性好（不易过拟合）

**3. 训练完整性**
- 虽然 Exp4 只训练了 67/80 epochs
- 但 Loss 已经很低（47）
- SRΨ 需要 79 epochs 才达到 108.89

### 为什么 Transformer 表现最差？

**1. 过度建模**
- Self-attention 建模**长程依赖**
- Burgers 方程主要是**局部相互作用**
- 计算开销大，收益小

**2. 训练困难**
- 需要仔细调参（学习率、warmup）
- 容易陷入局部最优
- Loss 波动大（1277-1299）

**3. 位置编码**
- 相对位置编码在此任务上可能不适合
- 周期性边界条件需要特殊处理

## 💡 结论与建议

### **核心结论**

1. ✅ **R 算子是 SRΨ 的关键组件**
   - 移除导致 60%+ 性能损失
   - 必须保留

2. ⚠️ **SRΨ 在 Burgers 方程上不如简单 Conv**
   - Conv 更简单、更快、更准确
   - SRΨ 可能是**过度设计**

3. ❌ **Transformer 不适合此任务**
   - 性能最差
   - 训练最慢
   - 过度复杂

### **改进建议**

#### **短期改进**（针对 SRΨ）

1. **简化 S 算子**
   - 参考纯卷积的设计
   - 减少参数量
   - 提高训练效率

2. **轻量化架构**
   - 减少隐藏层维度（64 → 32）
   - 减少层数（3 → 2）
   - 目标：参数量 < 100K

3. **改进训练策略**
   - 添加验证集监控
   - Early stopping
   - Learning rate schedule

#### **长期研究**

1. **扩展到更复杂的任务**
   - 2D/3B 方程
   - 多物理场耦合
   - 更长的时间预测范围

2. **探索 SRΨ 的优势领域**
   - 需要长程依赖的问题
   - 多尺度现象
   - 非线性波动

3. **混合架构**
   - Conv + SRΨ 结合
   - 局部用 Conv，非局部用 SRΨ
